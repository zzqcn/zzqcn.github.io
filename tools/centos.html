网络配置失败,显示Failed to start LSB: Bring up/down networking¶

假设要配置的接口是ens33,

  1. 确认/etc/sysconfig/network-scripts/ifcfg-ens33 中的MAC地址 配置正确, 即HWADDR的值

  2. 如果HWADDR正确还报此错误, 运行:

    systemctl stop NetworkManager
systemctl disable NetworkManager

  3. 运行service network restart即可

共享CentOS虚拟机上的文件给Windows本机访问¶

以下操作都需要root权限.

  1. yum install samba

  2. 编辑 /etc/samba/smb.conf, 内容如下:

    [global]
        workgroup = WORKGROUP
        security = user

        passdb backend = tdbsam

        printing = cups
        printcap name = cups
        load printers = yes
        cups options = raw

[dev]
        path = /home/zzq
        public = yes
        writable = yes
        create mask = 0644
        directory mask = 0755

    配置重点:

    • [global]中的workgroup要设成和本机工作组一致.
    • [dev]是共享的虚拟目录名, 从windows访问时将看到名为dev的共享目录
    • path为要共享的实际目录
    • public = yes, 允许所有用户访问, 如果只允许某些用户访问, 则使用 valid users = aaa bbb 这样的设置, aaa,bbb为用户
    • writable = yes, 允许写操作
    • create mask = 0644, 设置新建文件权限, 使得新建文件不带有可执行属性

  3. 设置用户访问密码. 这里为了方便, 没有创建新用户, 直接使用已有用户zzq:

    smbpasswd -a zzq

  4. 关闭SELinux.

    • 临时关闭 setenforce 0;
    • 永久关闭, 修改/etc/selinux/config, 将 SELINUX=enforcing 改为 SELINUX=disabled , 重启即可

注解

如果不想关闭SELinux, 可以运行 setsebool -P samba_enable_home_dirs=1 开启samba用户HOME目录的权限, 共享其他目录及权限设置方法请查询selinux+samba.

  1. 在防火墙上开放samba所用端口:

    firewall-cmd --permanent --add-port=139/tcp
firewall-cmd --permanent --add-port=445/tcp
systemctl restart firewalld

  2. 启动samba服务, 并设置为开机启动:

    systemctl enable smb.service
systemctl restart smb.service

  3. 在windows上通过 \\<虚拟机IP> 这样的地址在资源管理器访问即可

VMware虚拟机配置静态IP¶

虚拟机配置为DHCP非常方便, 但是会出现ip地址改变的问题, 导致要经常修改远程登录工具如SecureCRT中的IP地址. 为了解决这个问题, 可以对虚拟机设置静态IP地址. 步骤如下:

  1. 在VMware Workstation中打开虚拟网络编辑器, 这里我们使用VMnet8, 它是类型是NAT模式, 注意把”使用本地DHCP服务将IP地址分配给虚拟机”复选框前的打勾取消, 并确认子网IP和掩码.

  2. 点击”NATR设置”按钮, 确认子网信息, 设置网关IP

  3. 本机通过虚拟网卡也加入了这个虚拟网络VMnet8, 因此我们需要配置这个虚拟网卡的IP. 打开系统设置中网络适配器设置, 找到这块虚拟网卡(一般名字是VMware Network Adapter VMnet8), 配置其IPv4属性:

  4. 启动CentOS虚拟机, 使用管理员权限编辑 /etc/sysconfig/network-scripts/ifcfg-ens33 文件, 按如下配置进行修改(注意不同虚拟机可能网卡名字不同, 这里是ens33):

    TYPE=Ethernet
BOOTPROTO=static
ONBOOT=yes
IPADDR=192.168.128.4
GATEWAY=192.168.128.1
NETMASK=255.255.255.0
